## ⚙️ 2. Configuration (1 Minute)

Open the **`.env`** file with a text editor and set your credentials:

```bash
DB_PASSWORD=my_secure_password
PASSWORD=my_admin_erp_password
LANGUAGE=es-ES     # or en-US (UI tdc.bat) 
TRYTON_LANGUAGE=es # or fr,de (TRYTON MODULES)

```

*The system uses these values to build the database and the admin user automatically.*

---

## 🚀 3. Deployment (3 Minutes)

1. **Open a Terminal:** Navigate to the `TryDockCmd` folder.
2. **Run the Manager:**
```cmd
tcd.bat

```


3. **The Pre-Flight Check:** The script will check your Docker status and load the language.
4. **Automatic Install:** Select **Option 0 (Install)**.
* It will pull images.
* It will build the `tryton` (Production) and `tryton_demo` (Lab) databases.
* It will activate 41 core modules and apply the legislation layer.



---

## 🖥️ 4. Access Tryton (1 Minute)

Once the audit shows **[OK]** for all modules:

1. Select **Option 10** in the menu.
2. The manager will verify Port 8000 and launch your browser at `http://localhost:8000`.
3. **Login:** User: `admin` | Password: (The one you set in `.env`).

---

## 🛠️ 5. Basic Operations

* **Need to stop?** Option 3 (`stop.bat`).
* **Need to check health?** Option 1 (`status.bat`).
* **Errors?** Option 5 (`errors.bat`) to see the last 24h critical trace.

---

### 🛡️ Pro-Tips

* **Logs:** Check the `/log` folder for full traceability.
* **Backups:** Use Option 6 frequently. Files are stored in `/backup`.
* **Updates:** To update to the latest Tryton version (e.g., 7.8.5), simply change the image tag in `compose.yml` and run Option 0 again. **TryDockCmd is version-aware!**
